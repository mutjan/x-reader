## 复查任务
对每条新新闻，判断：
1. 当前分组归属是否正确（如果已归入事件）
2. 是否有更匹配的已有事件
3. 是否应创建新事件组

同时检查每个高相似度事件内的已有新闻：如果某条已有新闻实际上不属于该事件的核心范围，也应提出调整建议。

## 输出格式
只返回需要修正的条目。如果分组正确，返回空数组 []。

严格的JSON数组，每个元素包含：
{
  "news_id": "新闻的id字段",
  "current_event_id": "当前归属事件的group_id（如该新闻不在任何事件中则为null）",
  "suggested_event_id": "建议归属事件的group_id，或 'new_event' 表示应创建新事件组",
  "reason": "修改原因，说明为什么需要调整"
}

注意事项：
- news_id 必须严格使用新闻条目的 id 字段值
- suggested_event_id 使用 'new_event' 特殊值表示建议创建新事件组
- 只返回需要修正的条目，分组正确的新闻不出现在结果中
- 确保返回严格的JSON数组，不要任何其他文字
