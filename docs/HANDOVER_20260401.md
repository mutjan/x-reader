## 🔄 标准工作流程
### 完整数据处理流程（推荐）
```bash
# 1. 拉取最新新闻并生成处理提示词
python main.py --time-window 24

# 输出会包含:
# - 快照ID: abc12345
# - 基础处理提示词文件: temp/ai_prompt_abc12345.txt
# - 快照文件: .snapshots/snapshot_abc12345.json

# 2. 处理基础提示词
# 将ai_prompt_abc12345.txt内容发送给Claude/其他AI，将返回结果保存为 _ai_base_result.json

# 3. 导入基础结果，生成分打提示词
python scripts/import_results.py --snapshot-id abc12345 --base-result-file _ai_base_result.json

# 输出会包含:
# - 打分提示词文件: temp/scoring_prompt_abc12345.txt

# 4. 处理打分提示词
# 将scoring_prompt_abc12345.txt内容发送给Claude/其他AI，将返回结果保存为 _ai_scoring_result.json

# 5. 导入打分结果，完成发布
python scripts/import_results.py --snapshot-id abc12345 --continue --scoring-result-file _ai_scoring_result.json
```

### 快捷流程（仅需基础信息不需要打分）
```bash
# 1. 拉取新闻生成提示词
python main.py --time-window 24

# 2. 处理提示词得到基础结果
# 保存为 _ai_base_result.json

# 3. 仅导入基础结果（不会发布，仅保存中间状态）
python scripts/import_results.py --snapshot-id abc12345 --base-result-file _ai_base_result.json
```

## 🎯 Phase 2 核心变更
### 1. AI处理能力增强
- **基础处理输出**: 中文标题、摘要、新闻类型、扩展分析、核心实体
- **打分处理输出**: 分级（S/A+/A/B/C）、综合评分（0-100）
- **特殊规则**:
  - Hinton相关内容自动+5分，最低85分
  - 时代情绪关键词自动加分
  - C级（<65分）内容自动过滤
  - 多维度评分逻辑：热度、独特性、读者价值、可延伸深度、扩展性

### 2. 工作流优化
- 快照机制升级：支持保存中间处理结果
- 断点续跑：使用`--continue`参数可从任意阶段继续处理
- 多批次处理：支持大数量分批处理，不会丢失上下文
- 自动清理：7天以上的旧快照和提示词自动清理

### 3. 前端功能完善
| 字段 | 展示方式 | 交互功能 |
|------|----------|----------|

| 类型 | 12种分类彩色标签 | 点击排序同类型新闻 |
| 标题 | AI生成中文标题 | - |
| 摘要 | 自动截断3行 | hover展开完整内容 |
| 实体 | 灰色标签 | 点击筛选包含该实体的新闻 |
| 扩展性 | 缩略展示 | hover弹出完整扩展分析 |

### 常见问题处理
#### 问题1: 导入结果时提示"URL匹配失败"
**原因**: AI返回结果中的URL与快照中的URL不一致
**解决**: 检查AI返回结果是否严格保留了original_url字段，不要修改URL内容

#### 问题2: 打分后新闻数量变少
**原因**: C级（<65分）的新闻会被自动过滤
**解决**: 正常逻辑，无需处理，低质量内容会被自动丢弃

#### 问题3: 快照找不到
**原因**: 快照超过7天会被自动清理
**解决**: 重新运行main.py生成新的快照

#### 问题4: 发布失败
**原因**: GITHUB_TOKEN权限不足或网络问题
**解决**: 检查token是否有仓库写入权限，或手动执行git push
