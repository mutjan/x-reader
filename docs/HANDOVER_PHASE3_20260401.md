## 📋 新增功能总览
Phase 3在Phase 2基础上新增三大核心能力：
1. ✅ **事件自动分组**：同事件新闻自动识别归并，生成时间线展示
2. ✅ **人工反馈机制**：支持前端直接修正新闻评分，反馈自动持久化
3. ✅ **智能校准系统**：基于历史反馈自动优化AI评分，持续提升准确性

## 🔄 完整工作流说明
### 1. 事件分组功能（自动生效）
**无需额外操作**，系统在每次发布时自动处理：
- 对所有新闻进行实体匹配+语义相似度分析
- 同事件的新闻自动归为一组，生成事件时间线
- 前端新增「事件追踪」标签页，展示所有热点事件
- 事件按重要性排序，展示事件发展脉络和所有相关报道

### 2. 人工反馈功能使用
#### 步骤：
1. 在新闻列表中找到需要修正的新闻
2. 点击分级标签旁的「↻」按钮，弹出修正对话框
3. 选择修正后的分级、输入修正分数（0-100），可填写修正原因
4. 提交后反馈即时在页面生效，下次发布时会自动应用到所有同源新闻

#### 说明：
- 人工修正优先级 **高于** AI自动评分
- 修正后的新闻会显示「✓」标记，标识为人工审核
- 所有反馈记录自动保存30天，用于后续AI校准

### 3. 自动校准功能使用
#### 手动执行校准：
```bash
# 仅生成校准报告，不修改数据
python scripts/calibrate_scores.py --dry-run

# 执行完整校准流程并发布
python scripts/calibrate_scores.py

# 仅校准最近3天的新闻
python scripts/calibrate_scores.py --days 3
```

#### 定时执行（建议配置）：
```bash
# 每日凌晨2点自动执行校准
0 2 * * * cd /path/to/x-reader && python scripts/calibrate_scores.py >> /var/log/calibration.log 2>&1
```

#### 校准逻辑：
- 基于最近30天的反馈数据生成校准规则
- 每天自动校准最近7天的新闻评分
- 规则有效期30天，自动过期避免过时影响
- 支持dry-run预览效果，所有操作自动备份可回滚

## 🔧 运维说明
### 数据备份
- 反馈数据自动备份：每次修改自动备份到 `data/feedback/backups/`
- 校准自动备份：每次校准自动备份原新闻数据到 `news_data.json.bak.calibration_*`
- 所有备份默认保留30天，自动清理过期文件

### 常见问题处理
#### 问题1: 事件分组不准确
**原因**: 相似度阈值不合适或实体匹配不足
**解决**: 可以调整 `src/processors/event_grouper.py` 中的参数：
- `entity_threshold`: 共同实体阈值，默认3
- `similarity_threshold`: 相似度阈值，默认0.85

#### 问题2: 校准后评分不符合预期
**原因**: 校准规则生成偏差
**解决**:
1. 使用 `--dry-run` 参数预览校准效果
2. 删除错误的反馈记录重新生成规则
3. 手动调整 `data/feedback/calibration_rules.json` 中的规则

#### 问题3: 反馈提交后没有生效
**原因**: 反馈需要在下次发布时才会全局生效
**解决**: 手动运行一次发布流程 `python main.py --no-fetch` 或等待定时发布

## 📊 功能验证方法
### 事件分组验证：
1. 运行主流程拉取新闻并发布
2. 访问前端页面，查看是否有「事件追踪」标签页
3. 确认同事件的新闻被正确归为一组，时间线顺序正确

### 反馈功能验证：
1. 点击任意新闻的修正按钮，提交修正
2. 确认页面显示修正后的评分和「✓」标记
3. 检查 `data/feedback/feedback_records.json` 中是否有对应记录
4. 重新发布后确认所有同源新闻都应用了修正

### 校准功能验证：
1. 提交至少5条同一实体的反馈（如都是关于OpenAI的新闻）
2. 运行 `python scripts/calibrate_scores.py --dry-run`
3. 确认生成了对应实体的校准规则
4. 查看校准报告，确认相关新闻的评分按规则调整
