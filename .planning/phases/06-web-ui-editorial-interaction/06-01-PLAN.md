  <name>Task 1: 扩展后端/api/news接口，支持筛选和排序参数</name>
  <files>src/web/app.py</files>
  <action>
    1. 在/api/news接口中新增filter和sort参数解析：
       - filter参数：JSON格式，包含domain（领域）、score_min（最低分）、score_max（最高分）、start_time（开始时间）、end_time（结束时间）、ratings（评级列表）字段
       - sort参数：格式为"field:order"，支持字段：score（热度）、published_at（时间）、sources_count（来源数），order支持asc/desc
    2. 实现筛选逻辑（多个条件为"与"关系，per D-02）：
       - 领域筛选：匹配news.type == filter.domain
       - 热度筛选：news.score >= filter.score_min 且 news.score <= filter.score_max
       - 时间筛选：news.published_at >= filter.start_time 且 news.published_at <= filter.end_time
       - 评级筛选：news.rating in filter.ratings
    3. 实现排序逻辑（per D-04）：
       - 默认排序为published_at:desc（per D-05）
       - 根据sort参数指定的字段和方向排序
    4. 不修改现有业务逻辑，只在返回前增加筛选和排序步骤
  </action>

    </automated>
  </verify>
  <done>
