# Plan 02-02: Implement import workflow for scoring results

## Goal
Extend the import_results.py script to handle both base processing results and scoring results, and integrate the complete AI processing flow.

## Background
The import_results.py script currently only handles loading base AI processing results, not scoring results. The scoring step is a separate workflow that needs to be integrated after base processing is complete.

## Requirements
1. AI-01: Multi-dimensional score generation
2. AI-02: Automatic classification into predefined technology subdomains
3. AI-03: Accurate content summarization

## Implementation Steps

### Step 1: Extend import_results.py command line arguments
Add new arguments:
- `--scoring-result-file`: Path to scoring result JSON file (optional, if provided runs scoring step)
- `--continue`: Continue processing from existing base processing results in snapshot

### Step 2: Implement two-phase import flow
Modify import_results.py to support:
1. Phase 1: Import base processing results → save intermediate processed items to snapshot
2. Phase 2: Import scoring results → apply scores and grades to intermediate items
3. Combined flow: Import both base and scoring results in one command

### Step 3: Integrate AIScorer.load_manual_scoring_result()
Add call to AIScorer.load_manual_scoring_result() after base processing is complete to:
- Parse scoring JSON response
- Apply scores and grades to processed items
- Apply special scoring rules (Hinton bonus, zeitgeist boost, etc.)
- Filter out C-grade items

### Step 4: Fix bugs in import_results.py
Fix the undefined variables in lines 77, 126, 127, 128:
- Add `duplicate_remover = DuplicateRemover()` initialization
- Fix reference to `all_raw_items` and `unique_items` variables

## Success Criteria
- import_results.py can import base processing results and save intermediate state
- import_results.py can import scoring results and apply them to intermediate items
- Scoring rules (special bonuses, filters) are correctly applied
- C-grade items are properly filtered out
- All undefined variable bugs are resolved
- Workflow produces fully processed items with complete AI fields
