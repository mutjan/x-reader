# Plan 02-01: Integrate AI scoring into manual processing workflow

## Goal
Extend the manual AI processing workflow to support both base content processing and scoring in a single flow.

## Background
The AIScorer class already exists in ai_processor.py with complete scoring logic, but it's not integrated into the manual processing flow. Currently the manual processor only generates prompts for base processing (title, summary, type, entities), not for scoring.

## Requirements
1. AI-01: Multi-dimensional score generation (heat, novelty, domain match, timeliness)
2. AI-02: Automatic classification into predefined technology subdomains
3. AI-03: Accurate content summarization

## Implementation Steps

### Step 1: Modify ManualProcessor.process_batch() method
Update the ManualProcessor.process_batch() method in `src/processors/ai_processor.py` to:
- After generating the base processing prompt and snapshot, also generate the scoring prompt template
- Save both prompts with the same snapshot ID in the temp directory
- Log instructions for both processing steps

### Step 2: Update snapshot structure
Extend the snapshot JSON structure to include:
- Status field indicating whether base processing and scoring are completed
- Fields to store intermediate results

### Step 3: Add result persistence helpers
Add methods to the ManualProcessor class to:
- Save intermediate base processing results to the snapshot
- Load intermediate results for scoring step

## Success Criteria
- Running `python main.py` generates both `ai_prompt_<snapshot_id>.txt` (base processing) and `scoring_prompt_<snapshot_id>.txt` (scoring) files
- Both prompts reference the same snapshot ID and contain the correct news items
- Log output clearly explains the two-step processing workflow
