# Plan 02-03: Extend output schema and frontend integration

## Goal
Ensure all AI processing results are correctly exposed in the output JSON and frontend interface.

## Background
The ProcessedNewsItem model already contains all the necessary AI fields, but we need to verify they are properly serialized and passed to the frontend.

## Requirements
1. AI-01: Multi-dimensional score generation
2. AI-02: Automatic classification into predefined technology subdomains
3. AI-03: Accurate content summarization

## Implementation Steps

### Step 1: Verify output serialization
Check `ProcessedNewsItem.to_dict()` and `ProcessedNewsItem.to_frontend_dict()` methods in `src/models/news.py` to ensure:
- All AI fields (chinese_title, summary, grade, score, news_type, extension, entities) are included in output
- Type mapping is correct for news types
- Score and grade are properly formatted for frontend consumption

### Step 2: Update frontend index.html
Verify the frontend HTML template in `src/publishers/github_pages.py` (or the static template file) correctly displays:
- AI-generated Chinese titles
- Summaries
- Grades (S/A+/A/B) with appropriate visual styling
- Scores
- News types (product, funding, etc.)
- Entity tags
- Extension analysis

### Step 3: Add validation and error handling
Add validation steps in the processing pipeline to:
- Check for missing required AI fields before publishing
- Ensure scores are within valid range (0-100)
- Ensure grades are one of the allowed values (S/A+/A/B/C)
- Log warnings for items with incomplete AI data

## Success Criteria
- All AI-generated fields are present in the final output JSON
- Frontend correctly displays all AI information with proper styling
- Invalid/missing AI data is properly handled before publishing
- End-to-end workflow successfully produces news items with complete AI processing results
