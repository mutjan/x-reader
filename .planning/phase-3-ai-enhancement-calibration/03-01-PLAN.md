# Plan 03-01: Implement same-event news automatic grouping

## Goal
实现同事件新闻的自动识别、归并和时间线展示功能。

## Background
当前系统每条新闻独立展示，相同事件的多篇报道分散在列表中，用户无法看到事件全貌。需要基于实体匹配+语义相似度的混合模式，自动将同事件新闻归为一组。

## Requirements
AI-04: Related news reports on the same event are automatically grouped into event timelines

## Implementation Steps

### Step 1: Implement similarity calculation utilities
在`src/utils/`下新增`similarity.py`工具类：
- 实现Jaccard相似度算法用于实体集合匹配
- 实现余弦相似度算法用于标题/摘要文本匹配
- 封装统一的`calculate_news_similarity(news1, news2)`接口，返回0-1的相似度值
- 配置可调整的阈值参数：实体匹配阈值≥3个相同核心实体，整体相似度阈值≥0.85

### Step 2: Add event grouping processor
在`src/processors/`下新增`event_grouper.py`模块：
- 实现`group_news(news_items)`方法，接收ProcessedNewsItem列表，返回分组后的事件列表
- 分组逻辑：先按实体过滤候选集，再计算文本相似度，符合阈值的归为同一事件
- 每个事件组包含：事件ID、事件主标题（取最高评分新闻的标题）、新闻列表（按发布时间升序）、覆盖实体列表、最高评分
- 保留时间最早的新闻作为事件起始点，最新的作为事件最新进展

### Step 3: Extend data model and output schema
- 在`ProcessedNewsItem`中新增`event_id`可选字段，标记所属事件
- 新增`Event`数据类，包含事件相关字段
- 在输出JSON中新增`events`字段，包含所有事件组数据
- 保持原有新闻列表结构不变，向后兼容

### Step 4: Update frontend to support event timeline display
修改`index.html`：
- 新增事件分组展示区域，在新闻列表上方展示重要事件
- 事件卡片展示事件标题、相关新闻数量、时间跨度、最高分级
- 点击事件展开时间线，按时间顺序展示该事件的所有相关新闻
- 支持按事件热度/时间排序

## Success Criteria
- 相同事件的3篇及以上报道可以被正确识别并归为一组
- 事件时间线按发布时间正确排序，展示事件发展脉络
- 不同事件的新闻不会被错误归为一组
- 原有独立新闻展示功能不受影响
- 处理100条新闻的分组耗时≤1秒
