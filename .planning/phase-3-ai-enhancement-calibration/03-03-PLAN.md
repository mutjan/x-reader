# Plan 03-03: Implement offline scoring calibration mechanism

## Goal
基于历史反馈数据自动生成评分校准规则，定期批量校准新闻评分，持续提升AI打分准确性。

## Background
人工反馈数据积累后，需要通过分析这些反馈自动优化评分规则，减少后续重复的人工修正工作。采用每日离线批量校准的模式，避免实时调整带来的评分波动。

## Requirements
AI-05: System supports manual feedback on AI scores to continuously improve evaluation accuracy

## Implementation Steps

### Step 1: Implement calibration rule generator
在`src/processors/`下新增`score_calibrator.py`模块：
- 实现`CalibrationRuleGenerator`类，负责分析反馈数据生成校准规则
- 分析维度：实体关联偏差、新闻类型偏差、关键词偏差、来源偏差
- 规则格式示例：
  ```json
  {
    "rules": [
      {
        "id": "rule_001",
        "type": "entity",
        "value": "OpenAI",
        "score_adjustment": +3,
        "confidence": 0.92,
        "sample_count": 12,
        "expires_at": "2026-04-30T00:00:00"
      }
    ]
  }
  ```
- 规则置信度要求：至少5个以上相同类型的反馈样本才能生成规则，置信度≥0.8
- 规则有效期：默认30天，过期自动失效

### Step 2: Implement calibration execution engine
在`score_calibrator.py`中实现`CalibrationEngine`类：
- 实现`apply_calibration(news_item, rules)`方法，对单条新闻应用校准规则
- 校准逻辑：多个规则可以叠加，但总分不能超过100，不能低于0
- 重新计算分级的逻辑与原有逻辑保持一致
- 保留原始AI评分和校准后的评分，便于审计

### Step 3: Implement daily calibration script
在`scripts/`下新增`calibrate_scores.py`脚本：
- 执行流程：
  1. 加载近30天的反馈数据
  2. 生成/更新校准规则
  3. 加载近7天的所有新闻数据
  4. 应用校准规则重新计算所有新闻的评分和分级
  5. 保存更新后的新闻数据
  6. 生成校准报告：调整新闻数量、平均分变化、规则有效性统计
- 输出校准报告到`data/feedback/calibration_report_YYYYMMDD.md`
- 脚本支持`--dry-run`模式，仅生成报告不实际修改数据

### Step 4: Add scheduling support
- 在定时执行脚本中增加每日凌晨2点自动执行校准的任务
- 校准完成后自动触发重新发布到GitHub Pages
- 校准过程中保留历史数据备份，支持回滚

## Success Criteria
- 基于至少5条相同类型的反馈可以自动生成有效的校准规则
- 规则应用后相关新闻的评分向用户预期方向调整
- 每日校准可以在10分钟内完成（≤1000条新闻）
- 校准过程中数据不丢失，可回滚
- 校准报告清晰展示调整效果和规则有效性
- 校准规则30天后自动失效，避免过时规则影响评分
