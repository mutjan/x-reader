# Plan 03-02: Implement manual scoring feedback mechanism

## Goal
实现用户对AI评分的手动修正和反馈功能，为后续校准提供数据基础。

## Background
当前AI评分完全基于固定规则，无法根据编辑的人工判断进行调整。需要支持用户在前端直接修正新闻的评分和分级，反馈数据自动持久化存储。

## Requirements
AI-05: System supports manual feedback on AI scores to continuously improve evaluation accuracy

## Implementation Steps

### Step 1: Implement feedback storage layer
在`src/data/`下新增`feedback_store.py`模块：
- 实现`FeedbackStore`类，负责反馈数据的读写、备份
- 存储路径：`data/feedback/feedback_records.json`
- 自动备份机制：每次写入前自动备份到`data/feedback/backups/feedback_YYYYMMDD_HHMMSS.json`，保留最近30天的备份
- 实现CRUD接口：添加反馈、查询反馈、按新闻ID查询、按实体查询

### Step 2: Add feedback API endpoints
修改`src/web/app.py`（如果存在），或在发布流程中支持静态反馈数据提交：
- 新增POST `/api/feedback`接口，接收用户反馈数据
- 验证数据格式：news_id必须存在，corrected_grade必须是S/A+/A/B/C，corrected_score必须在0-100之间
- 自动记录反馈时间、用户IP（可选）
- 返回操作结果状态

### Step 3: Implement frontend feedback interaction
修改`index.html`：
- 在每条新闻的评级标签旁新增"修正"按钮，点击弹出反馈对话框
- 对话框支持选择修正后的分级、输入修正分数、填写修正原因
- 提交后显示成功/失败提示
- 已反馈的新闻显示标记，区分AI原始评分和人工修正评分
- 修正后的评分即时在页面生效

### Step 4: Integrate feedback into processing pipeline
修改`src/publishers/github_pages.py`的发布流程：
- 发布前加载反馈数据，对有修正的新闻直接使用用户修正后的评分和分级
- 修正后的评分优先级高于AI自动评分
- 在输出JSON中标记哪些新闻是人工修正过的
- 前端显示时用不同样式区分人工修正的评分

## Success Criteria
- 用户可以在前端对任意新闻提交评分修正反馈
- 反馈数据自动持久化存储，自动备份
- 修正后的评分和分级优先于AI原始评分展示
- 已反馈的新闻有明确的视觉标记
- 反馈数据格式正确，包含所有必要字段
- 反馈提交响应时间≤1秒
